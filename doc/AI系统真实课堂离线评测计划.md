# AI系统真实课堂离线评测模板包

## 0. 这份文档解决什么问题
这份模板包用于验证：
1. 你们的 AI 系统在真实课堂中是否起效。
2. 起效体现在哪些维度（学生学习、教师备课、课堂互动）。
3. 哪些场景无效或反向影响，如何做下一轮优化。

文档内已包含：
1. 实验怎么做（分组、流程、采集节奏）。
2. 指标有什么意义（为什么看这个指标）。
3. 指标如何统计（公式、统计检验、口径）。
4. 指标结果出来后怎么处理（结论、分级决策、版本迭代）。
5. 文档末尾可直接复制的 CSV 表头。

---

## 1. 实验总设计（先从小规模真实课堂开始）

### 1.1 实验目标
1. 验证 AI 建议是否能提升课堂学习效果。
2. 验证 AI 模拟是否能命中真实学生困惑与错误。
3. 验证系统是否能节省教师备课时间并提高教案可执行性。

### 1.2 分组方案
建议采用同教师内对照，减少教师个体差异。

1. 对照组：教师按原方式备课上课。
2. 实验组A：使用快速模式建议后改教案上课。
3. 实验组B：使用深度模式建议后改教案上课。
4. 可选消融组：深度模式关闭记忆强化/遗忘，用于验证机制增益。

### 1.3 推荐样本规模（MVP）
1. 教师数：8到12位。
2. 每位教师课次数：至少4节（2节对照+2节实验）。
3. 总课次数：32到48节。
4. 学科：先从1到2门开始（如数学、语文）。

---

## 2. 实验详细过程（按时间顺序执行）

### 阶段A：准备阶段（第1周）
1. 固定版本：模型版本、提示词版本、画像版本、记忆机制开关。
2. 选课题池：每学科选 8 到 12 个教学主题，保证对照与实验可配对。
3. 教师培训：统一如何使用系统、如何记录课堂数据、如何填评分表。
4. 数据脱敏：学校、班级、学生、教师统一匿名编码。

### 阶段B：预采集（第2周）
1. 采集基线：课前小测正确率、历史同主题成绩、常见错误。
2. 试跑 3 到 5 节课：检验采集流程是否顺畅、字段是否缺失。
3. 修订模板：把试跑中不好填的字段优化成可执行版本。

### 阶段C：正式执行（第3到6周）
1. 每节课执行顺序：
	1. 记录课堂样本主表。
	2. 按组别进行备课（对照或实验）。
	3. 课堂实施并记录观察项。
	4. 进行课后小测。
	5. 教师填写打分表。
2. 每周一次质检：检查缺失值、异常值、字段口径一致性。

### 阶段D：分析复盘（第7到8周）
1. 统计计算：按指标公式计算每节课结果。
2. 显著性检验：验证组间差异是否可信。
3. 失败案例复盘：至少整理 5 个失败样本。
4. 输出报告：形成可决策结论和下一轮优化清单。

## 3. 采集模板（字段与用途）

### 3.1 课堂样本主表（记录课堂元信息）
用途：把每节课的“人、课、模型、输入、输出”串起来，作为后续统计主键。

| 字段名 | 类型 | 必填 | 说明 | 示例 |
|---|---|---|---|---|
| 样本编号 | 文本 | 是 | 每节课唯一编号 | MATH_G7_001 |
| 学校编号 | 文本 | 是 | 匿名学校ID | SCH_03 |
| 教师编号 | 文本 | 是 | 匿名教师ID | T_12 |
| 班级编号 | 文本 | 是 | 匿名班级ID | C_07 |
| 学科 | 枚举 | 是 | 学科名称 | 数学 |
| 年级 | 枚举 | 是 | 年级 | 七年级 |
| 课题 | 文本 | 是 | 主题名称 | 一元一次方程 |
| 上课日期 | 日期 | 是 | yyyy-mm-dd | 2026-05-20 |
| 分组类型 | 枚举 | 是 | 对照组/实验组A/实验组B/消融组 | 实验组B |
| 使用模式 | 枚举 | 是 | 快速模式/深度模式/不使用 | 深度模式 |
| 模型提供商 | 文本 | 否 | 如 qwen/openai | qwen |
| 模型名称 | 文本 | 否 | 模型ID | qwen-plus |
| 提示词版本 | 文本 | 是 | 追踪版本 | pv_20260520_01 |
| 画像版本 | 文本 | 是 | 追踪版本 | prof_v3 |
| 记忆机制配置 | 文本 | 否 | 强化/遗忘参数摘要 | reinforce_on_decay_086 |
| 输入材料类型 | 枚举 | 是 | txt/docx/pptx/pdf/manual | pdf |
| 解析后文本字数 | 数值 | 是 | 输入文本长度 | 3850 |
| 解析状态 | 枚举 | 是 | 成功/空文本/失败 | 成功 |
| 解析备注 | 文本 | 否 | OCR或异常说明 | OCR后可用 |
| 运行是否成功 | 布尔 | 是 | 是否产出结果 | 是 |
| 输出报告编号 | 文本 | 否 | 报告ID | REP_20260520_45 |
| 教案建议采纳率 | 数值 | 否 | 0到1之间 | 0.60 |
| 备注 | 文本 | 否 | 其他影响因素 | 临时换教室 |

### 3.2 教师打分表（单节课）
用途：衡量“是否好用、是否有效、是否愿意继续用”。

基础字段：样本编号、教师编号、填写时间、分组类型。

评分字段（1到5分）：
1. 建议可执行性评分。
2. 学情匹配度评分。
3. 困惑点命中评分。
4. 错误预测命中评分。
5. 备课省时感知评分。
6. 课堂效果改进评分。
7. 总体价值评分。

选择题字段：
1. 是否继续使用。
2. 推荐意愿分（0到10分）。
3. 主要帮助环节。

开放题字段：
1. 最有价值建议。
2. 最不准确预测。
3. 下一步改进建议。

### 3.3 指标计算明细表（分析主表）
用途：把“样本表+老师表+学生测评”汇总后直接做统计。

| 指标名 | 含义 | 统计口径 |
|---|---|---|
| 课前小测正确率 | 教学前知识掌握水平 | 每节课课前小测平均正确率 |
| 课后小测正确率 | 教学后知识掌握水平 | 每节课课后小测平均正确率 |
| 正确率提升值 | 学习增益核心指标 | 课后小测正确率 - 课前小测正确率 |
| 关键知识点达成率 | 重点目标掌握程度 | 达成学生数/总学生数 |
| 困惑点召回率 | 模拟对真实困惑的覆盖能力 | 命中困惑点数/真实困惑点总数 |
| 错误模式召回率 | 模拟对真实错误的覆盖能力 | 命中错误模式数/真实错误模式总数 |
| Top3命中率 | 前三预测的实用性 | 前3预测命中数/真实问题数 |
| 课堂参与变化值 | 学生互动改善程度 | 实验课参与度 - 对照课参与度 |
| 教师可执行性均分 | 建议落地能力 | 建议可执行性评分均值 |
| 教师总体价值均分 | 产品综合价值 | 总体价值评分均值 |
| 净推荐值 | 教师传播意愿 | 推荐者占比 - 贬损者占比 |
| 备课时长 | 教师时间成本 | 单节课备课分钟数 |
| 备课时长变化值 | 时间收益 | 本次备课时长 - 教师个人基线 |
| 建议采纳率 | 建议被采用程度 | 采纳条数/建议总条数 |
| 输出稳定性方差 | 重复运行稳定程度 | 同输入多次运行关键结果方差 |
| 评审一致性系数 | 人工评判可靠性 | Kappa/ICC |
| 异常样本标记 | 数据质量控制 | 人工或规则标记 |

## 4. 这些指标分别是干什么的

### 4.1 学习效果类
1. 正确率提升值：直接回答“学生是否学会了”。
2. 关键知识点达成率：验证是否命中教学目标。

### 4.2 模拟真实性类
1. 困惑点召回率：验证 AI 是否真正看到了学生卡点。
2. 错误模式召回率：验证 AI 对典型错因的识别能力。
3. Top3命中率：判断教师最先看到的建议是否有用。

### 4.3 教师效率与体验类
1. 备课时长变化值：验证是否节省时间。
2. 教师可执行性均分：验证建议是否可直接落地。
3. 净推荐值：验证教师是否认可长期使用价值。

### 4.4 工程可靠性类
1. 输出稳定性方差：防止结果随机漂移。
2. 评审一致性系数：确保打分不是“看人下菜”。

## 5. 怎么统计这些指标（可直接执行）

### 5.1 数据清洗
1. 去除运行失败样本。
2. 去除解析状态为失败且无补录文本的样本。
3. 缺失值处理：
	1. 必填字段缺失直接剔除。
	2. 评分缺失超过30%的教师样本单独标记。

### 5.2 统计口径
1. 每节课为一个样本单位。
2. 教师内配对比较优先（同一教师对照 vs 实验）。
3. 结果至少按“总体、学科、年级”三层展示。

### 5.3 统计检验
1. 配对 t 检验：适合近似正态指标（如正确率提升值）。
2. Wilcoxon 符号秩检验：适合非正态或样本较小。
3. Bootstrap 置信区间：输出 95% 区间，增强可信度。
4. 效应量：建议报告 Cohen's d 或 Cliff's delta。

### 5.4 最低报告要求
1. 每个核心指标给出：均值、标准差、样本量、p值、效应量、95%置信区间。
2. 给出至少 5 个失败样本分析，不只报平均好结果。

## 6. 拿到指标后怎么处理（决策规则）

### 6.1 版本是否通过
建议设三档：
1. 通过：
	1. 正确率提升值显著为正。
	2. 困惑点召回率和错误模式召回率均达到目标阈值。
	3. 教师可执行性均分 >= 4.0。
2. 条件通过：
	1. 学习效果有提升但不显著。
	2. 教师体验指标达标。
	3. 需继续扩大样本复验。
3. 不通过：
	1. 核心学习指标无提升或下降。
	2. 异常样本比例高。
	3. 教师负反馈集中。

### 6.2 问题归因与优化优先级
1. 输入问题优先修：OCR失败、学科错配、材料长度不足。
2. 再修模拟问题：困惑点漏检、错因泛化、层级区分不明显。
3. 再修交互问题：建议太长、不可执行、缺课堂语境。

### 6.3 下一轮迭代产出
每轮必须产出：
1. 指标变化对比表（本轮 vs 上轮）。
2. 失败案例清单（含根因与修复状态）。
3. 版本变更说明（改了什么，期望提升什么指标）。

## 7. 实验报告结构（可直接套用）

### 7.1 报告目录
1. 项目目标与验证假设。
2. 实验设计与样本分布。
3. 数据质量与清洗说明。
4. 指标定义与统计方法。
5. 结果总览（图表）。
6. 关键结论（按学科/年级分层）。
7. 失败案例复盘。
8. 版本决策与下轮计划。
9. 附录（问卷、表头、公式）。

### 7.2 每章最少要写什么
1. 目标与假设：明确本轮要证明什么。
2. 设计与样本：写清分组逻辑与样本量。
3. 数据质量：写清剔除规则和剔除数量。
4. 指标与统计：写清公式、检验方法、阈值。
5. 结果：先总后分层，必须含置信区间。
6. 结论：明确是否通过，不允许模糊表述。
