# AI系统真实课堂简版评测问卷

这是一份给真实课堂试用时使用的轻量版评测材料。

如果你觉得详细版评测计划太复杂，可以先只用这一份。它的目标不是做严格科研，而是先快速回答一个更实际的问题：

1. 老师用了这个系统之后，备课和上课是不是更有帮助。
2. 学生能不能感受到老师讲课方式有变化。
3. 这个变化到底是正向的、无感的，还是负向的。

建议用法：

1. 先找 3 到 5 位老师试用 2 到 4 周。
2. 每位老师至少选 2 节使用本系统辅助备课的课。
3. 每节课后让老师填 1 份教师问卷。
4. 每节课后抽样让学生填 1 份学生问卷。
5. 最后只看几个最简单的结论：是否有效、有效体现在哪、问题主要在哪。

---

## 1. 最简单的执行方式

### 1.1 老师怎么做
1. 用系统提前做一次课前预演。
2. 根据系统建议调整自己的教案或讲法。
3. 上完课后，花 3 分钟填写教师问卷。

### 1.2 学生怎么做
1. 不需要知道系统细节。
2. 只需要在课后花 2 分钟填写学生问卷。
3. 问卷主要关注：老师这节课是否更清楚、更容易懂、是否更关注学生困难。

### 1.3 管理者怎么做
1. 收集教师和学生问卷。
2. 统计每个题目的平均分。
3. 看开放题里最常出现的正面反馈和负面反馈。

## 2. 教师问卷

填写说明：

请老师在每次使用系统辅助备课并完成授课后填写一次。

评分规则：

1 分 = 完全不同意
2 分 = 比较不同意
3 分 = 一般
4 分 = 比较同意
5 分 = 完全同意

### 2.1 基础信息

| 字段 | 填写内容 |
|---|---|
| 教师编号 |  |
| 学科 |  |
| 年级 |  |
| 课题 |  |
| 上课日期 |  |
| 本节课是否使用本系统辅助备课 | 是 / 否 |
| 使用的是哪种模式 | 快速模式 / 深度思考模式 / 不清楚 |

### 2.2 教师量表题

请按 1 到 5 分打分：

1. 这个系统帮助我更快发现了这节课的教学风险。
2. 这个系统给出的学生反应预测对我有参考价值。
3. 这个系统帮助我提前发现了学生可能出现的困惑点。
4. 这个系统帮助我提前发现了学生可能出现的常见错误。
5. 这个系统给出的建议比较容易直接用于改教案。
6. 使用这个系统后，我这节课的备课更有方向感。
7. 使用这个系统后，我这节课讲解重点更清楚。
8. 使用这个系统后，我更容易照顾到基础薄弱学生。
9. 使用这个系统后，我认为课堂效果有提升。
10. 我愿意在以后的备课中继续使用这个系统。

### 2.3 教师开放题

1. 这次系统最有帮助的一点是什么？
2. 这次系统最不准确的一点是什么？
3. 你是否真的根据系统建议改了讲课方式？如果有，改了什么？
4. 你觉得这个系统最适合用在哪类课上？
5. 你最希望它下一步改进什么？

## 3. 学生问卷

填写说明：

请在课后让学生匿名填写，不需要写姓名。建议只保留班级、学科、日期即可。

评分规则：

1 分 = 完全不同意
2 分 = 比较不同意
3 分 = 一般
4 分 = 比较同意
5 分 = 完全同意

### 3.1 基础信息

| 字段 | 填写内容 |
|---|---|
| 班级编号 |  |
| 学科 |  |
| 年级 |  |
| 课题 |  |
| 上课日期 |  |

### 3.2 学生量表题

请按 1 到 5 分打分：

1. 这节课老师讲得比平时更清楚。
2. 这节课老师更容易讲到我原本会不懂的地方。
3. 这节课老师给的例子或解释更容易理解。
4. 这节课老师更像提前知道我们可能会错在哪里。
5. 这节课我更愿意跟着老师一起思考。
6. 这节课我更敢提问或回答问题。
7. 这节课结束后，我觉得自己对知识点理解更清楚。
8. 我希望以后老师继续用这种方式讲课。

### 3.3 学生开放题

1. 这节课老师哪里讲得特别清楚？
2. 这节课老师有没有提醒到你原本容易错的地方？如果有，是什么？
3. 这节课你还有哪里没听懂？
4. 如果以后继续这样上课，你最希望老师保持什么？

## 4. 最简单的统计办法

如果你不想做复杂统计，只做下面 4 件事就够了。

### 4.1 看教师平均分
把教师量表题每题都算一个平均分。

建议重点看这 4 题：

1. 是否帮助发现学生困惑。
2. 是否帮助发现学生错误。
3. 是否容易改教案。
4. 是否愿意继续使用。

简单判断规则：

1. 平均分大于等于 4.0：说明整体认可。
2. 平均分在 3.0 到 4.0：说明有帮助，但还不稳定。
3. 平均分低于 3.0：说明当前价值不明显。

### 4.2 看学生平均分
建议重点看这 4 题：

1. 老师讲得是否更清楚。
2. 是否更容易讲到学生不懂的地方。
3. 是否更像提前知道学生会错在哪里。
4. 学生是否希望以后继续这样讲课。

简单判断规则和教师问卷一致：

1. 大于等于 4.0：学生明显感知到正向变化。
2. 3.0 到 4.0：变化存在，但不够强。
3. 低于 3.0：学生感知不明显。

### 4.3 看开放题高频词
把老师和学生开放题里的反馈归成几类，例如：

1. 讲解更清楚。
2. 更能提前提醒易错点。
3. 更照顾基础弱学生。
4. 建议太空泛。
5. 和真实课堂不完全一致。

如果正面反馈反复出现，说明系统方向基本对。
如果负面反馈集中在同一问题，说明下一轮优化优先级已经很明确。

### 4.4 最后只写一个简单结论
可以直接按下面格式写：

1. 老师是否觉得有效：有效 / 部分有效 / 暂不明显。
2. 学生是否感知到变化：明显感知 / 有一定感知 / 基本无感。
3. 最主要的帮助：例如“提前发现困惑点”“讲解更有层次”。
4. 最主要的问题：例如“建议不够具体”“部分预测不准”。
5. 是否建议继续试点：建议 / 有条件建议 / 暂不建议。

## 6. 如果你想再更简单一点

如果连量表都觉得多，可以只保留下面 3 个问题。

### 给老师
1. 这个系统对这节课有没有帮助？
2. 最大帮助是什么？
3. 最大问题是什么？

### 给学生
1. 这节课老师讲得是否更容易懂？
2. 老师有没有提前提醒你容易错的地方？
3. 你希望以后继续这样上课吗？

这已经足够做一轮非常轻量的试点验证。
