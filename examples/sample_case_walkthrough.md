# 示例案例：七年级数学《一元一次方程》

这是一份给新手使用的软件演示案例。你可以直接用这个案例快速理解系统怎么输入、怎么分析、怎么看结果。

## 案例目标

让第一次使用软件的老师在5分钟内完成一次完整体验：

1. 导入教学材料
2. 设置学科、年级和模型参数
3. 生成学生反应模拟
4. 阅读优化建议
5. 查看 AI 修订后的新教案
6. 导出报告

## 推荐使用方式

### 方式一：直接上传示例文件

上传文件：examples/sample_input_linear_equation.txt

### 方式二：复制文本到输入框

把示例文件中的全文复制到“或直接粘贴教案/逐字稿文本”区域。

## 界面填写示例

- 学科：数学
- 年级：七年级
- 课题：一元一次方程
- 模型提供商：DeepSeek / Qwen / GLM / OpenAI / Gemini / Claude / Kimi / MiniMax 任意一个
- API Key：填写你自己的密钥
- Base URL：使用系统默认值即可
- Model：使用系统默认值即可
- 报告导出格式：建议先选 Markdown 或 HTML

## 建议观察的重点

运行后，你可以重点看以下几个区域：

### 1. 关键知识点

系统一般会识别出类似内容：
- 一元一次方程
- 等式性质
- 移项
- 合并同类项
- 列方程
- 符号变化
- 检验

### 2. 学生反应模拟

你会看到5类学生画像，例如：
- 基础薄弱型：更容易在移项、符号变化、等号保持上出错
- 中等偏下型：能跟随例题，但遇到变式题容易停顿
- 中等稳定型：能完成常规题，但应用题列方程可能不稳
- 中等偏上型：能较快完成标准题，但细节表达可能不够规范
- 拔高拓展型：会希望看到更高阶或更开放的问题

### 3. 教学优化建议

系统大概率会给出类似建议：
- 在“移项”环节加入更明确的依据解释，而不只是讲“移过去变号”
- 在例题和练习之间增加一题过渡练习，降低中等偏下学生掉队概率
- 在应用题教学前加入“找等量关系”的单独支架步骤
- 给学有余力的学生追加两道开放性或变式挑战题
- 将课后作业明确分成基础、提升、拓展三层

### 4. AI修订后的新教案

系统现在还会根据前面的学生反应和优化建议，直接生成一版新的教案或课堂流程稿。你可以重点看：

- 导入环节是否更清楚
- 关键例题是否拆得更细
- 是否增加了过渡练习
- 是否补上了分层任务
- 是否更照顾基础薄弱和中下学生

### 5. AI做了哪些修改

系统现在还会给出一组简短摘要，告诉老师它到底改了什么。比如：

- 补强了导入环节的铺垫
- 增加了移项易错点提醒
- 在例题和练习之间加入了过渡练习
- 为不同层次学生补了分层任务

### 6. 原教案与修订后教案对照

界面会左右并排显示“原教案”和“修订后教案”，适合新手直接比较：

- 哪些地方被保留了
- 哪些地方被补充了
- 哪些环节被重写了
- 教学节奏是否更清晰了

## 新手操作路径

1. 启动应用
2. 选择“数学”与“七年级”
3. 上传 examples/sample_input_linear_equation.txt
4. 填写模型 API 信息
5. 点击“开始预演与优化”
6. 查看结果区的“关键知识点”“学生反应模拟”“教学优化建议”“AI做了哪些修改”“原教案与修订后教案对照”
7. 下载报告

## 这个案例适合说明什么

这个案例特别适合向第一次接触系统的老师说明三件事：

1. 软件不是替老师上课，而是帮老师做课前预演
2. 软件关注的是不同层次学生可能在哪里卡住
3. 软件最终目的是帮助老师优化教案、练习和作业分层

## 如果你要做演示

推荐现场演示时这样说：

“我先把一份七年级数学教案丢进系统，系统不会直接评价老师讲得好不好，而是会从不同学生层次出发，预测谁可能在移项、列方程、应用题这些地方掉队，然后再给出可执行的改进建议。”
