# AI虚拟学生教学预演系统
# 新手操作说明书（GUI图形界面版）

本说明书面向第一次接触本软件的老师，目标是：
- 不写代码，也能完成一次完整预演
- 看懂界面每个模块是做什么的
- 学会申请并配置 DeepSeek / Qwen 的 API Key

说明：界面和按钮名称以软件当前版本为准，若后续版本有微调，请以界面实际文字为准。

---

## 1. 这个软件是做什么的？

这是一款“课前预演”工具。

你可以上传教案、逐字稿、PPT、PDF（或直接粘贴文本），软件会模拟不同层次学生可能出现的：
- 困惑点
- 提问点
- 常见错误

并给出：
- 教学优化建议
- 修订后教案
-（深度模式）课堂分模块讲解后学生的反馈以及裁决汇总结果

一句话理解：
先在电脑里“演一遍课”，再去上课。

---

## 2. 第一次使用前你需要准备什么？

### 2.1 电脑和网络
- Windows 电脑
- 能访问大模型服务官网（用于申请 API）
- 软件运行时建议保持联网

### 2.2 一个可用的大模型 API Key
本软件当前通过 API 调用模型，大模型是软件的大脑，为了访问这个大脑，我们需要先配置 API Key。

推荐新手先用：
- DeepSeek (推荐)
- 或Qwen

申请步骤见第 9 章和第 10 章。

---

## 3. 启动软件后，先看哪里？

打开软件后，先看左侧侧边栏最上方：
- `模型API设置`
- `打开API设置窗口`

请先配置 API，再做其他操作。

如果没有配好 API，主界面的“开始预演与优化”按钮会被禁用，这是正常保护机制。

---

## 4. 一次完整使用流程（手把手）

### 步骤 1：配置 API（最先做）
1. 点击侧边栏第一行 `打开API设置窗口`
2. 在弹窗中选择 `模型提供商`（例如 deepseek 或 qwen，推荐deepseek更便宜）
3. 粘贴 `API Key`
4. 检查/填写 `Base URL`（默认值一般可用）
5. 选择 `Model`
6. 点击 `保存设置`
7. 关闭窗口

看到侧边栏显示“API Key：已配置”，再进入下一步。

### 步骤 2：设置教学信息
在侧边栏 `输入设置` 中：
1. 选择 `学科`
2. 选择 `年级`
3. 如需微调学生画像，点击 `打开学生配置窗口`

### 步骤 3：输入教学材料
主界面可二选一或混合使用：
1. 上传文件（支持 txt / md / docx / pptx / pdf）
2. 直接在文本框粘贴内容

说明：
- 图像识别文字功能已默认开启（对图片型 PDF/PPT 更友好）
- 上传多个文件时会自动合并文本

### 步骤 4：选择输出格式和分析模式
1. 选择 `学生反应报告格式`（Markdown / HTML / Word）
2. 选择 `修订后教案格式`（Markdown / HTML / Word）
3. 选择分析模式：
- `快速模式`：速度快、调用少，适合先摸底
- `深度思考模式`：多轮讨论，更细致

### 步骤 5：开始预演
点击 `开始预演与优化`。

运行中：
- 深度模式会显示进度条和阶段提示
- 结束后会展示整套结果

### 步骤 6：查看和下载结果
结果区域支持：
- 查看学生反应、难度评估、建议、修订教案
- 下载学生反应报告
- 下载修订后教案
-（输入为单个 PPTX 时）下载 AI 修订建议版 PPT

## 5. 界面模块说明（做什么、怎么用）

## 5.1 模型API设置
作用：连接大模型服务。

你会看到：
- 模型提供商
- API Key
- Base URL
- Model

怎么用：
- 第一次配置后点“保存设置”
- 换模型或换服务商时再打开修改

常见提醒：
- API Key 里不要带中文、换行、全角符号
- 如果提示额度不足，请先充值或提升配额

## 5.2 输入设置（学科/年级）
作用：告诉系统“当前课属于哪个学科和年级”。

怎么用：
- 按真实授课场景选择
- 年级会影响学生画像和分析结果

## 5.3 当前学科学生配置
作用：配置“虚拟学生群体”。

你可以：
- 增减学生
- 改姓名
- 调层级（基础薄弱到拔高拓展）
- 改优势/薄弱点/常见错误/支持需求
- 导入导出模板

建议：
- 不熟悉时先用默认模板
- 跟班级差异大时再做微调

## 5.4 材料输入区
作用：提供待分析内容。

支持：
- txt / md / docx / pptx / pdf
- 手动粘贴文本

建议：
- 文本尽量完整（目标、重难点、例题、练习）
- 如果是 PPT，尽量提供可编辑文本版（不是纯图片）

## 5.5 分析模式
### 快速模式
适合：
- 时间紧
- 先看整体风险

特点：
- 一次调用
- 出结果快

### 深度思考模式
适合：
- 重点课
- 需要看“分模块”问题

特点：
- 教师讲稿 + 多轮学生讨论 + 裁决汇总
- 结果更细，但耗时更长

## 5.6 结果区
主要看三类信息：
1. 整课汇总（关键知识点、反应、难度、建议）
2. 深度模式分模块细节
3. 下载区（报告、修订教案、PPT建议版）

---

## 6. 深度模式里的四个关键词（通俗版）

深度模式下，老师会把整个课堂分模块讲解，在每个模块结束都会经历两轮讨论，讨论内容包括：
1) 学生互相质疑与补充（指出误解、遗漏、过度自信）
2) 反方挑战者专门找出当前讲解最可能导致误解的点
3) 教学观察员裁决，给出可执行修正意见

在“讨论裁决结果”中你会看到：

### 模块共识
- 含义：大家都同意的结论
- 用法：可优先执行

### 即时调整
- 含义：这节课现在就能改的动作
- 用法：直接转成课堂操作

### 模块分歧
- 含义：意见不一致、还不确定的点
- 用法：提醒你谨慎判断

### 记忆更新
- 含义：学生状态变化，影响后续模块
- 用法：看学习困难是否被解决或反复出现

## 8. 常见问题与处理

### Q1：按钮是灰的，点不了“开始预演与优化”
原因：API 未配置完整。
处理：
1. 打开API设置窗口
2. 填写并保存 API Key / Base URL / Model

### Q2：报“额度不足/余额不足”
原因：模型账户欠费或配额用完。
处理：
1. 登录模型平台检查余额
2. 充值或提升配额
3. 再重试

### Q3：报“UnicodeEncodeError / latin-1”
原因：API Key 里混入中文、换行或特殊字符。
处理：
1. 重新复制纯净 key
2. 不要带空格、换行、说明文字
3. 保存后重试

### Q4：点下载后页面重刷，看起来结果没了
说明：
- 新版本会自动保留最近一次结果并重新显示
- 属于 Streamlit 重跑机制，不影响下载内容

### Q5：上传 PPT 后导出内容是什么？
- 原始材料区：PPT 提取文本
- 修订后教案区：AI 基于提取内容生成

## 9. DeepSeek API Key 申请教程（新手版）

以下步骤以 DeepSeek 官方平台为准，如页面有变动请按官网提示操作。

### 9.1 注册与登录
1. 打开 DeepSeek 官方开放平台（https://platform.deepseek.com/api_keys）
2. 注册账号并完成登录

### 9.2 实名/安全设置（如平台要求）
1. 按页面提示完成认证
2. 绑定手机或开启安全验证

### 9.3 开通 API 服务并充值
1. 进入控制台的 Billing / 计费 / 余额页面
2. 充值（或开通可用额度）

### 9.4 创建 API Key
1. 进入 API Keys 页面
2. 点击“创建新密钥”
3. 复制并妥善保存（通常只显示一次）

### 9.5 在软件中填写
1. 打开软件 `模型API设置` 窗口
2. 模型提供商选 `deepseek`
3. 粘贴 API Key
4. Base URL 使用默认值（通常可直接用）
5. 选 Model（如 deepseek-v4-flash 或 deepseek-v4-pro）
6. 点击 `保存设置`

推荐给新手：
- 先用 flash（更快、成本低）
- 重点课再切 pro（更强）

## 10. Qwen API Key 申请教程（新手版）

Qwen 通常通过阿里云百炼（DashScope）体系使用。以下以阿里云控制台流程为准。

### 10.1 注册与登录
1. 打开阿里云官网并登录 (https://bailian.console.aliyun.com/)
2. 进入百炼 / DashScope 相关控制台

### 10.2 开通服务与计费
1. 开通百炼模型服务
2. 完成计费方式设置（按平台要求充值或绑定支付）

### 10.3 创建 API Key
1. 进入 API Key 管理页
2. 创建新 Key
3. 复制并保存

### 10.4 在软件中填写
1. 打开 `模型API设置` 窗口
2. 模型提供商选 `qwen`
3. 粘贴 API Key
4. Base URL 使用默认值（兼容模式地址）
5. 选 Model（qwen-turbo / qwen-plus / qwen-max）
6. 点击 `保存设置`

新手选型建议：
- qwen-turbo：速度快，先摸底
- qwen-plus：较均衡，常规备课
- qwen-max：能力更强，复杂课例

## 11. 给新手老师的实操建议

### 建议 1：先“快后深”
- 先跑快速模式看全局
- 再用深度模式查关键模块

### 建议 2：每次只改少量画像
- 先小改（1~2个学生）看结果变化
- 有效果再做全班模板调整

### 建议 3：把 AI 结果当“预演参考”
- 不替代教师判断
- 用于提前发现风险、提高备课效率
